  - 警告は「アクセシビリティで Koecho をオンにして再試行」を通常手順とし、すでにオンなら現在の Koecho を追加し直すこと、それでも反映されない場合のみ再起動することを案内する。再起動を必須手順とは書かない
  - 回帰テストで権限エラー時のクリップボード復元と、その後の同一パネル再確定が元の対象へ貼り付け、成功時の状態と履歴記録まで正しく進むことを検証する。macOS の実権限反映は手動確認する
- [ ] InputPanelScriptStrip にスクリプトの drag-to-reorder を追加する
  - macOS 27 の `.reorderable()` + `.reorderContainer(for:)` で List 以外（横 ScrollView の HStack）でも並び替えが可能になった。機能自体は増えず「どこでできるか」が変わる: 設定画面を開かずに入力パネル上でスクリプト順を変えられる
  - デプロイターゲット macOS 14 のままなら `if #available(macOS 27, *)` ガードが必要。deployment target を上げた後にやれば分岐なしで書ける
  - 実装する時に、管理画面 List の `onMove`（`ReplacementRuleManagementView.swift:52` / `ScriptManagementView.swift:24`）を `reorderable` に揃えるかも同時に判断する
- [ ] 選択範囲へ適用できる内蔵テキスト操作を追加する
  - スクリプト設定で種類を「カスタムスクリプト」または「内蔵機能」から選べるようにする。カスタムスクリプトでは従来どおりコマンドを設定し、内蔵機能ではユーザー向けの機能名だけを選択する。内蔵スクリプトの名前・パス・実装詳細は表示しない
  - 初期の内蔵機能は `Indent`、`Unindent`、`Quote` とする。既存のスクリプトボタン、ショートカット、並び順と同じ実行導線で扱う。選択範囲を持たない確定時 Auto-run の対象にはしない
  - `Indent` / `Unindent` は機能を 2 spaces 用・4 spaces 用に分けず、インデント幅を 2 または 4 spaces から設定する。`Unindent` は同じ幅を上限として行頭の spaces を除去する。同じ機能を幅違いで複数登録でき、`Indent (2 spaces)` のように設定を含む機能ラベルで区別する
  - `Quote` は選択範囲と交差する各行の先頭へ Markdown の `> ` を付ける。`>` のみを使う別モードは設けず、異なる形式が必要な場合はカスタムスクリプトで対応する
  - 行操作後も対象行を選択した状態に保ち、同じ機能を連続実行できるようにする。選択がない場合はカーソル行を対象にする。選択範囲の維持はユーザー設定にせず、内蔵行操作の一貫した挙動とする
  - 内蔵実装はアプリ Bundle に含め、ユーザーによる別途ダウンロード、実行権限の付与、パス設定を不要にする。保存時は Bundle 内の絶対パスではなく安定した機能 ID とユーザー設定を保持し、アプリの移動・更新後も解決できるようにする
  - 文書先頭の Indent を含め、変換で追加・保持する先頭・末尾の空白を失わないようにする。現行 `ScriptRunner` の stdout trim をそのまま内蔵行操作へ適用せず、既存カスタムスクリプトの出力挙動も意図せず変更しない
  - 選択範囲は `NSRange` の UTF-16 座標で渡されるため、Unicode を含むテキストでも対象範囲と変換後の選択範囲がずれないことをテストする。volatile テキストを確定してから実行し、プログラム由来の `NSTextView` 更新では callback suppression を維持する
