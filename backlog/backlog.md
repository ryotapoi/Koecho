  - アクセシビリティ権限エラー時に `frontmostApplication` が失われ、復帰した入力パネルからは同じ対象へ再試行できない。テキストと貼り付け先を保ったままエラー表示に戻す
  - 失敗からの復帰では、パネル表示時に捕捉した対象アプリを再取得しない。System Settings 等を開いていても元の `NSRunningApplication` を保持し、次の確定でその同一対象へ貼り付ける。対象アプリが終了していた場合は「対象アプリは終了した」と表示し、再試行ではなくキャンセルして開き直す
  - 復帰時に音声認識は再開しない。確定済みテキスト、対象アプリ、エラーだけを復元し、成功・Escape・空確定では従来どおり対象アプリを破棄する

  - 回帰テストで権限エラー時のクリップボード復元と、その後の同一パネル再確定が同一の対象アプリへ貼り付け、成功時の状態と履歴記録まで正しく進むことを検証する。macOS の実権限反映は手動確認する
- [ ] InputPanelScriptStrip にスクリプトの drag-to-reorder を追加する
  - macOS 27 の `.reorderable()` + `.reorderContainer(for:)` で List 以外（横 ScrollView の HStack）でも並び替えが可能になった。機能自体は増えず「どこでできるか」が変わる: 設定画面を開かずに入力パネル上でスクリプト順を変えられる
  - デプロイターゲットは macOS 14 のままとし、macOS 27 以降だけ `if #available(macOS 27, *)` で有効化する。macOS 14〜26 では入力パネルの並び替えを提供せず、既存の設定画面での並び替えを使う
  - `ReorderDifference` は Script の ID を基準に既存の `scripts` 配列へ適用し、確定した並びだけを直ちに永続化する。外部アプリとの drag-and-drop、複数選択、項目の結合はこの項目に含めない
  - 設定画面の List は既存の `onMove` を維持する。入力パネルと同じ API へ揃えるリファクタリングは、両画面を同時に改善する必要が生じた時の別タスクとする
  - macOS 27 SDK でのビルドと実機の横スクロール領域での drag-to-reorder を手動確認し、設定を再起動後も順序が保たれることを自動テストする
- [ ] 選択範囲へ適用できる内蔵テキスト操作を追加する
  - スクリプト設定で種類を「カスタムスクリプト」または「内蔵機能」から選べるようにする。カスタムスクリプトでは従来どおりコマンドを設定し、内蔵機能ではユーザー向けの機能名だけを選択する。内蔵スクリプトの名前・パス・実装詳細は表示しない
  - 内蔵機能の表示名は `Decrease Indent`、`Increase Indent`、`Block Quote` とする。入力パネルではそれぞれ SF Symbols の `decrease.indent`、`increase.indent`、`text.quote` をボタンに使い、英語名はツールチップとアクセシビリティラベルに表示する。設定画面ではアイコンと英語名を併記する
  - 内蔵項目はユーザーが編集できる名前・コマンドではなく、安定した機能 ID と設定値で識別する。カスタム項目と同じ名前でも重複とは見なさず、既定の内蔵項目との重複だけを防ぐ
  - 機能導入後の初回起動時に、この3機能を既存のユーザースクリプトの後ろへ `[Decrease Indent] [Increase Indent] [Block Quote]` の順で既定登録する。`Increase Indent` / `Decrease Indent` の初期幅は 2 spaces とする。既存のユーザースクリプトは保持し、内蔵機能を重複登録しない
  - 初期登録済みであることを scripts 配列の空判定ではなく専用の永続フラグで管理する。ユーザーが既定項目を削除しても次回起動時に復元せず、必要なら設定画面から再登録できることを回帰テストする
  - 既存の Script 保存データは、新しい種類を持たない場合にカスタムスクリプトとして読み替える。新形式への保存後も、既存ユーザースクリプトの ID、順序、名前、コマンド、ショートカット、prompt 設定を失わない
  - 内蔵機能は既存のスクリプトボタン、ショートカット、並び順と同じ実行導線で扱う。確定時 Auto-run の候補からは常に除外する
  - `Increase Indent` / `Decrease Indent` は機能を 2 spaces 用・4 spaces 用に分けず、インデント幅を 2 または 4 spaces から設定する。`Decrease Indent` は同じ幅を上限として行頭の spaces を除去する。同じ機能を幅違いで複数登録でき、`Increase Indent (2 spaces)` のように設定を含む機能ラベルで区別する
  - `Block Quote` は選択範囲と交差する各行（空行を含む）の先頭へ Markdown の `> ` を付ける。すでに quote 済みの行でも実行ごとに 1 つ追加する。`>` のみを使う別モードは設けず、異なる形式が必要な場合はカスタムスクリプトで対応する
  - 非空の選択は、選択終端が次行の先頭に一致する場合を除き、選択と交差する各行を対象にする。選択がない場合はカーソル行を対象にする。行操作後は変換後の対象行全体（末尾の改行は除く）を選択し、同じ機能を連続実行できるようにする。これはユーザー設定にしない
  - 内蔵実装はアプリ Bundle に含め、ユーザーによる別途ダウンロード、実行権限の付与、パス設定を不要にする。保存時は Bundle 内の絶対パスではなく安定した機能 ID とユーザー設定を保持し、アプリの移動・更新後も解決できるようにする
  - 文書先頭の `Increase Indent` を含め、変換で追加・保持する先頭・末尾の空白を失わないようにする。現行 `ScriptRunner` の stdout trim をそのまま内蔵行操作へ適用せず、既存カスタムスクリプトの出力挙動も意図せず変更しない
  - 選択範囲は `NSRange` の UTF-16 座標で渡されるため、Unicode を含むテキストでも対象範囲と変換後の選択範囲がずれないことをテストする。空行、文書先頭、改行境界、入れ子 quote、2/4 spaces の不足インデント、既存カスタムスクリプトの保存データからの移行も回帰対象とする。volatile テキストを確定してから実行し、プログラム由来の `NSTextView` 更新では callback suppression を維持する
